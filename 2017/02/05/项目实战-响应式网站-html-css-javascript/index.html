前期准备

响应式网站概念

不同层面体验，媒体查询概念出现

  • flexible grid layout 弹性网格布局
  • flexible image 弹性图片
  • media queries 媒体查询

响应式网站是一个设计理念，它是多项技术的综合体。

响应式网站的有点

  • 减少工作量
    • 网站、设计、代码、内容只需要一份
    • 多出来的工作量只是JS脚本、CSS样式做一些改动
  • 节省时间
  • 每个设备都能得到正确的设计
  • 搜索优化
  • 更好的用户体验等

响应式网站的缺点

  • 会加载更多的样式和脚本资源
  • 设计比较难精确定位和控制
  • 老版本浏览器兼容性不好

媒体查询

CSS2

<link rel="stylesheet" type="text/css" href="site.css" media="screen" />

CSS3

CSS3进一步增强媒体查询，媒体类型 函数，匹配一些显示规则

  • all：应用于所有媒体类型
  • 表达式：()
  • 逻辑操作符：not and or only
  • {}前的语句是真 or 假：表示是否执行 {}
and：与 ；全部匹配
or（,） ： 或；任一条件匹配一个

@media not screen and (color),print and (color)
等同于：@media not screen and (color) or print and (color)

not：非，整个的媒体查询

示例01
@media not all and (monochrome) {...}
等同于@media not (all and （monochrome)){...}

示例02
@media not screen and (color), print and (color)
等同于
@media not screen and (color) or print and (color)
等同于
@media (not (screen and (color))), print and (color)

说明：not的有效范围只是到达 ,

only：仅仅，只有防止老旧的浏览器，不支持带媒体属性的查询，而应用到给定的样式。

media="only screen and (min-width: 401px) and (max-width: 600px)"
说明：就浏览器解析为 media="only"，这样设置后，当老浏览器不支持时，不会对执行语句中的规则

media="screen and (min-width: 401px) and (max-width: 600px)"
说明：旧浏览器解析为 media="screen"，这样设置后，老浏览器会执行 screen 设置，而screen后面的语句不会被执行

综上所述：如果为了达到兼容老浏览器，那么最好设置 only 属性

CSS3 媒体属性简洁

CSS3 新增了很多媒体属性

  • width：视口宽度
  • height：视口高度
  • device-width：渲染表面的宽度，就是设备屏幕的宽度
  • device-height：渲染表面的高度，就是设备屏幕的高度
  • orientation：检查设备处于横向还是纵向
  • aspect-ratio：基于视口宽度和高度的宽高比。width/height 如：16/9，4/3
  • device-aspect-ratio：渲染表面的宽度，就是设备屏幕的宽度。
  • color：每种颜色的位数 bits 如：min-color：16位，8位
  • resolution：检测屏幕或打印机的分辨率 如：min-resolution: 300dpi

注：以上属性都可以添加 min- 或 max- 前缀

viewport 视口

视口宽度 width

设备宽度 device-width

概念说明：

  • 桌面浏览器只有一个视口，就是网页内容区域
  • 移动设备，手机为例：
    • 布局视口（layout viewport），默认宽度很大；一般默认 960px；
    • 可视视口（visual viewport）：默认宽度是手机宽度；随着用户缩放页面不断变化的
    • 理想视口（ideal viewport）：把手机比作看电脑网页的放大镜；理想视口就是布局视口在一个设备上的最佳尺寸；理想视口下的页面便于浏览器浏览阅读。理想视口就是为构建手机浏览器优化的页面而添加的。

如何使用理想视口呢

<meta name="viewport" content="width=device-width" />

说明：如果不指定该属性，那么布局视口的宽度是厂商的默认值。

很多网站直接禁用了用户缩放，默认放大倍数为1。

比如：百度
<meta name="viewport" content="device-width",minimum-scale=1.0,maximum-scale=1.0,user-scalable=no" />
不仅设置了理想视口，还设置了

  • 用户默认缩放比例
  • 最小缩放比例
  • 最大缩放比例
  • 是否禁止用户缩放

px,em,rem

  • px（1个px相当于1个像素），绝对长度单位
  • em，相对长度单位
    • 相对参照物为 父元素 的font-size
    • 具有继承的特点
    • 当没有设置font-size时，浏览器会有一个默认的em设置：1em = 16px
  • rem的参照物为 根元素html，相当于参照物固定不变，所以比较好计算
    • 当没有设置font-size时，浏览器会有一个默认的rem，设置：1rem = 16px，这点与em是一致的
    • 用rem更安全
    • font-size: 62.5%，1rem =10px（10/16*100%），这样设置后，方便与px转换
    • font-size: 100%，1rem = 16px
    • 缺点是，并不是所有浏览器都支持，如ie8及以下版本

查看某个属性是否支持：http://caniuse.com/

当然解决兼容性，也很简单，看你需不需要兼容ie8

清除浮动 float

清除浮动，也就是闭合浮动

浮动是个令人又爱又恨的布局方式，用它会有很多问题

其实flex是个可以替代它的布局方式，但是需要考虑兼容性问题

浮动问题描述：class=”container”下的h1和p设置左浮动后，container容器高度为0，出现高度塌陷问题；且背景消失；在这里我们可以把容器子元素h1和p的浮动属性类比成浮云，对地面没有影响。在实际布局当中，这个问题不是我想要的。

解决方案01

在容器内，浮动元素最后，加入 <div style="clear:both"></div>。

虽然这个方案可以解决问题，但是我们在结构中添加了一个空标签，违背了结构和样式分离的web标准精髓。如果大量使用这种空标签，那么后期维护起来也是噩梦。

解决方案02

给浮动元素的父元素 class="container" 加入样式 overflow: auto; 或者 overflow: hidden;，浮动元素又回到了容器中；

解决方案03

给浮动元素的父元素class="container" 加入样式 float: left;

CSS3选择器

基本选择器

  • * 通配符选择器，选择所有元素，或者选则某个元素下的所有元素；

  • E 代表文档元素，比如 div p等

  • .class

  • #id

  • E F 后代选择器，也称为包含选择器

  • E > F 选择 E 元素的子元素 F，是 E 的直接子元素，不包含孙子元素及更往后的子元素

  • E + F 相邻元素选择器，共有相同的父元素，而且F在E的后面

li + li 会选择到后四个li

  • E ~ F

li + li 后四个元素都会被选择到

属性选择器

  • E[attr] 不论选择的属性是否有值都会被选择，如 a[href]
  • E[attr="value"]，这里是选择到完全等于”value”
  • E[attr^="value"]，选择开头值符合要求的，如 a[href^="https://"
  • E[attr$="value"]，选择结尾值符合要求的，如 img[src$=".png"
  • E[attr*="value"]，只要包含就可以
  • E[attr~="value"]，以空格分开的词汇，可以用这个选择器模糊选择到，如 <a title="Hello World"></a>，a[title~="Hello"或者a[title~="World"
  • E[attr|="value"]，选择以 value 或者 value- 开头的值
